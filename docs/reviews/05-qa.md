## Bugs found and fixed

### B1 [CRITICAL] — "Environment variable not found: DATABASE_URL" on Windows
**Root cause**: The `.env` file contained a hardcoded Linux path (`file:/home/z/my-project/db/custom.db`) that doesn't exist on Windows.
**Fix**: `db.ts` now resolves the path relative to the project root and auto-creates the `db/` folder. The `predev` hook runs `prisma db push` to create the DB file + tables before the server starts.

### B2 [HIGH] — `/api/settings` returns 500 when DB not initialized
**Root cause**: `getSettings()` calls `db.setting.findUnique()` which throws if the tables don't exist.
**Fix**: Wrapped `GET` and `POST` in try/catch — returns defaults + a warning instead of crashing.

### B3 [HIGH] — `/api/upload` returns 500 when DB not initialized
**Root cause**: Same as B2 — the upload route reads settings from the DB.
**Fix**: Wrapped the DB read in try/catch — falls back to the env var or default 2048MB limit.
